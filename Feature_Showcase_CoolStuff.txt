       → Direct maximum search within the band (ignores prominence)
       → Perfect for when you KNOW where the peak should be

    2. UNRANGED MODE: If no range specified
       → Global argmax across entire frequency band
       → File: multi_forward_worker.py, Line 189-191

SECONDARY PEAK DETECTION (Multiple peaks possible):
  • Configurable number: 0-10 secondary peaks (Default: 2)
  • TWO STRATEGIES PER SECONDARY:
    1. RANGED MODE: User specifies frequency band for each secondary
       → Maximum within band (bypasses prominence filter)
       → File: multi_forward_worker.py, Line 254-270
       
    2. UNRANGED MODE: Auto-detection via scipy.signal.find_peaks
       → Apply prominence threshold (Default: 0.5, Range: 0.0-10.0)
       → Apply amplitude threshold (Default: 2.0, Range: 0.0-100.0)
       → Exclude primary + already-claimed secondaries
       → Pick strongest remaining candidate
       File: multi_forward_worker.py, Line 271-279

CONFIGURATION:
  • Min Prominence: 0.0-10.0 (Default: 0.5)
    → Scipy find_peaks parameter for peak quality
    
  • Min Amplitude: 0.0-100.0 (Default: 2.0)
    → Absolute amplitude threshold to ignore tiny peaks
    
  • Per-Peak Frequency Ranges (Optional):
    → PRIMARY: constrain f0 to min-max band (e.g., 0.5-2 Hz)
    → UP TO 10 SECONDARY: each with independent range
    → Enable/disable each range independently
