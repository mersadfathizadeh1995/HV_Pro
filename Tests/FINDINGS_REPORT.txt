ISSUE #1: FFT with Empty Data Causes ZeroDivisionError
--------------------------------------------------------------------------------
SEVERITY: VERY HIGH (Application Crash)
LOCATION: hvsr_pro/processing/hvsr/spectral.py:51

DESCRIPTION:
The compute_fft() function crashes with ZeroDivisionError when given empty data.
This can occur if data loading fails silently or data slicing produces empty arrays.

REPRODUCTION:
    from hvsr_pro.processing.hvsr.spectral import compute_fft
    import numpy as np
    data = np.array([])
    frequencies, spectrum = compute_fft(data, 100.0, taper=None)
    # Raises: ZeroDivisionError: float division by zero

ROOT CAUSE:
    n = len(data)  # n = 0
    frequencies = rfftfreq(n, 1.0 / sampling_rate)  # 1.0/(n*d) = 1.0/0

SUGGESTED FIX:
    Add validation at start of compute_fft():
    if len(data) == 0:
        return np.array([]), np.array([])
