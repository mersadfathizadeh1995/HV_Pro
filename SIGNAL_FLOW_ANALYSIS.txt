================================================================================
COMPLETE SIGNAL FLOW & BUG ANALYSIS - BedrockMapping Settings System
================================================================================

SUMMARY OF ISSUES:
The settings system has 3 critical bugs preventing proper state synchronization:
1. NO MERGING of settings - Global Settings completely replaces state dicts
2. External changes to state are INVISIBLE to the Global Settings dialog
3. Both panel and dialog can have stale data references

================================================================================
SCENARIO A: USER CHANGES GLOBAL SETTINGS AND CLICKS OK
================================================================================

SIGNAL FLOW:
1. User modifies opacity slider in Global Settings dialog
   -> PyQt emits: opacity_spin.valueChanged signal
   -> Connected to: _save_current_contour() [line 206]

2. _save_current_contour() ONLY updates _contour_copy (local to dialog!)
   Location: global_settings.py lines 352-370
   Code:
   `
   self._contour_copy[key] = {
       'colorscale': self.cs_combo.currentText(),
       'opacity': self.opacity_spin.value(),
       'levels': self.levels_spin.value(),
       ...
   }
   `
   CRITICAL: NO SIGNAL EMITTED YET - state is unchanged!

3. User clicks OK button
   -> PyQt emits: btns.accepted signal
   -> Connected to: _on_ok() [line 83]

4. _on_ok() calls _write_to_state()
   Location: global_settings.py lines 442-444
   Code:
   `
   def _on_ok(self):
       self._write_to_state()
       self.accept()
   `

5. _write_to_state() writes changes to state AND emits signals
   Location: global_settings.py lines 422-437
   Code:
   `

   s.display_settings_changed.emit()  # LINE 436
   s.map_needs_refresh.emit()  # LINE 437
   `
   
   BUG #1 TRIGGER: Uses deepcopy() to REPLACE entire dict
   - This overwrites any external changes made to state
   - If layer_panel changed state while dialog was open, those changes are LOST

6. Signals propagate to all views
   - map_widget.py line 81: display_settings_changed -> _schedule_refresh()
   - view_2d.py line 93: display_settings_changed -> _schedule_render()
   - view_3d_plotly.py line 354: display_settings_changed -> _schedule()

7. Each view starts 300ms debounce timer
   - After 300ms delay, refresh() method is called
   - Each refresh reads state via get_contour_config()
   - get_contour_config() returns merged dict with current state values ✓

8. Views re-render with updated settings ✓

BUG #1: DEEPCOPY OVERWRITES LOST CHANGES
------
Problem: If user changes layer_panel before Global Settings dialog, those 
changes are stored in state. But when Global Settings writes:
  s.contour_layer_configs = deepcopy(self._contour_copy)
  
It COMPLETELY REPLACES the dict. _contour_copy doesn't have the layer_panel 
changes because _contour_copy was created from state at dialog __init__ time 
and was never updated with external changes.

Timeline:
1. state.contour_layer_configs['surface']['opacity'] = 0.7 (initial)
2. User opens Global Settings dialog
   _contour_copy = deepcopy(state) -> opacity=0.7 ✓
3. User closes dialog WITHOUT saving
4. User changes opacity in layer_panel to 0.5
   cfg = state.contour_layer_configs.get('surface')
   cfg['opacity'] = 0.5
   state NOW HAS opacity=0.5 ✓
5. User opens Global Settings dialog AGAIN
   NEW instance created
   _contour_copy = deepcopy(state) -> opacity=0.5 ✓
   But WAIT - if user already has Global Settings open from before:
   _contour_copy still has OLD value (0.7)
   When user clicks OK, it writes opacity=0.7 back to state
   The panel's change (0.5) is LOST!

BUG #2: _contour_copy NOT UPDATED FROM EXTERNAL STATE CHANGES
------
Problem: If Global Settings dialog is open and user changes layer_panel,
the dialog's _contour_copy dict is not updated. So dialog UI shows stale
values from when it was opened.

Timeline:
1. state.opacity = 0.7, state.levels = 15
2. User opens Global Settings dialog
   _contour_copy = deepcopy(state)
   UI shows opacity=0.7, levels=15 ✓
3. User doesn't touch Global Settings dialog
4. User changes layer_panel opacity to 0.5
   state.opacity = 0.5 ✓
5. User looks at Global Settings dialog
   Dialog still shows opacity=0.7 (STALE!)
   Because _contour_copy was never updated
6. User switches to BEDROCK tab in dialog
   UI loads bedrock values from _contour_copy['bedrock']
   Still hasn't refreshed SURFACE settings!
7. User clicks OK
   _write_to_state() writes _contour_copy
   Surface opacity=0.7 (stale) overwrites state opacity=0.5
   Panel's change is LOST!

FIX FOR SCENARIO A:
Instead of:
  s.contour_layer_configs = deepcopy(self._contour_copy)

Do:
  for layer_key in ('surface', 'bedrock', 'depth'):
      if layer_key in self._contour_copy:
          s.contour_layer_configs[layer_key].update(self._contour_copy[layer_key])

OR even better, add listener to display_settings_changed in the dialog to
refresh _contour_copy:
  self.state.display_settings_changed.connect(self._reload_all_from_state)

SCENARIO B: USER CHANGES OPACITY IN DISPLAY SETTINGS PANEL
================================================================================

SIGNAL FLOW:
1. User moves opacity slider in layer_panel
   Location: layer_panel.py line 163
   Code: self.opacity_slider.valueChanged.connect(self._on_disp_changed)

2. _on_disp_changed() gets reference to nested dict and modifies in place
   Location: layer_panel.py lines 340-351
   Code:
   `
   def _on_disp_changed(self, *_):
       if self._disp_loading:  # Guard
           return
       key = self.disp_layer.currentData()  # 'surface', 'bedrock', or 'depth'
       cfg = self.state.contour_layer_configs.get(key, {})  # GET REFERENCE
       cfg['opacity'] = self.opacity_slider.value() / 100.0  # MODIFY IN PLACE
       cfg['levels'] = self.levels_spin.value()
       cfg['colorscale'] = self.cs_combo.currentText()
       self.opacity_label.setText(...)
       self.state.display_settings_changed.emit()  # SIGNAL
       self.state.map_needs_refresh.emit()  # SIGNAL
   `
   
   BUG #2 TRIGGER: Gets reference then modifies directly
   - Works fine until Global Settings replaces the dict (see Scenario A step 5)

3. display_settings_changed signal sent
   All views connected: map_widget, view_2d, view_3d_plotly

4. Each view receives signal in order:
   - map_widget._schedule_refresh() - starts 300ms debounce
   - view_2d._schedule_render() - only if visible, starts 300ms debounce
   - view_3d_plotly._schedule() - only if visible, starts 300ms debounce

5. After 300ms delay (DEBOUNCE), first visible view refreshes
   map_widget.refresh() -> _build_folium_map()
   Reads: self.state.get_contour_config('surface')['opacity']
   Gets current value (0.5) ✓

6. User switches to 2D tab
   -> view_2d.showEvent() triggered
   -> _schedule_render() called
   -> Starts new 300ms debounce timer (could override existing!)

7. After 300ms, view_2d.render() executes
   Reads state again
   Gets current opacity=0.5 ✓

8. Similarly for 3D tab when activated

POTENTIAL ISSUE: 300ms DEBOUNCE CREATES STALE UI STATE
------
While debounce timer is running (up to 300ms), if user switches tabs:
- Old view hasn't rendered yet (still waiting on debounce)
- New view becomes visible
- New view's showEvent() triggers _schedule_render()
- But old view ALSO has a 300ms timer pending
- So you could have 2 renders queued with different timing

However, since both read from state AT RENDER TIME, they should get current values.

BUG #3: GLOBAL SETTINGS OVERWRITES PANEL CHANGES
------
Problem: After panel modifies state (opacity=0.5), if Global Settings dialog
writes to state, it can overwrite those changes with stale _contour_copy values.

Timeline:
1. state.opacity = 0.7 (initial)
2. User changes panel: opacity = 0.5
   cfg = state.contour_layer_configs['surface']  # Reference to dict
   cfg['opacity'] = 0.5
   state.opacity = 0.5 ✓
   Signals emitted
   Views schedule 300ms refresh
3. User opens Global Settings dialog
   _contour_copy = deepcopy(state)
   _contour_copy.opacity = 0.5 ✓
   _load_from_state() shows opacity=0.5 in dialog
4. User changes levels in dialog to 20 (doesn't touch opacity)
   _save_current_contour() reads ALL controls from UI:
   _contour_copy['surface'] = {
       'opacity': self.opacity_spin.value(),  # Reads from UI which shows 0.5
       'levels': self.levels_spin.value(),     # Reads from UI which shows 20
       ...
   }
   So _contour_copy is OK
5. User clicks OK
   _write_to_state() overwrites state with _contour_copy
   state.opacity = 0.5 ✓
   state.levels = 20 ✓

This SHOULD work! But there's a variant:

1. state.opacity = 0.7, levels = 15
2. User changes panel: opacity = 0.5 (leaves levels at 15)
   state.opacity = 0.5, state.levels = 15
3. Global Settings dialog ALREADY OPEN from before
   _contour_copy was created at __init__ when opacity=0.7
   Dialog UI shows opacity=0.7 (STALE!)
4. Without updating _contour_copy, user clicks OK
   _write_to_state(): s.contour_layer_configs = deepcopy(_contour_copy)
   state.opacity = 0.7 (overwrites the 0.5 from panel!)
   state.levels = 15 ✓

The panel's change is LOST because the dialog didn't know about it.

================================================================================
ROOT CAUSE SUMMARY
================================================================================

The settings system is "awfully messed up" because:

1. GLOBAL SETTINGS creates a local copy (_contour_copy) that:
   - Is initialized once from state at __init__
   - Is ONLY updated when user changes global_settings UI controls
   - Is NEVER refreshed if state changes externally (layer_panel changes)
   - Has NO listener for display_settings_changed signals

2. LAYER PANEL makes direct modifications to state dicts:
   - Gets reference: cfg = state.contour_layer_configs.get(key)
   - Modifies: cfg['opacity'] = value
   - Emits display_settings_changed signal
   - But if Global Settings replaces the entire dict, the reference becomes invalid

3. GLOBAL SETTINGS _write_to_state() REPLACES instead of MERGING:
   - s.contour_layer_configs = deepcopy(self._contour_copy)
   - This OVERWRITES any changes not in _contour_copy
   - If layer_panel changed opacity=0.5 but _contour_copy has 0.7, 
     the value reverts to 0.7 after Global Settings closes

4. NO REFRESH MECHANISM for dialog when state changes:
   - Global Settings dialog doesn't listen to display_settings_changed
   - So if panel changes state, dialog UI never updates to show new values
   - User thinks nothing happened and makes the same change again

================================================================================
RECOMMENDED FIXES
================================================================================

FIX #1: Make Global Settings merge, not replace
Location: global_settings.py line 429
Change from:
  s.contour_layer_configs = deepcopy(self._contour_copy)

To:
  for layer_key in self._contour_copy:
      if layer_key in s.contour_layer_configs:
          s.contour_layer_configs[layer_key].update(self._contour_copy[layer_key])
      else:
          s.contour_layer_configs[layer_key] = deepcopy(self._contour_copy[layer_key])

FIX #2: Make Global Settings dialog refresh from state on focus
Location: global_settings.py
Add method:
  def showEvent(self, event):
      super().showEvent(event)
      self._contour_copy = deepcopy(self.state.contour_layer_configs)
      self._legend_copy = deepcopy(self.state.layer_legend_configs)
      self._loading = True
      self._load_from_state()
      self._loading = False

OR connect to display_settings_changed signal:
  self.state.display_settings_changed.connect(self._reload_all_from_state)

FIX #3: Use blockSignals when layer_panel refreshes UI
Location: layer_panel.py _load_disp_layer() method
Change from:
  def _load_disp_layer(self, _idx=0):
      self._disp_loading = True
      ...
      self.opacity_slider.setValue(...)
      self._disp_loading = False

To:
  def _load_disp_layer(self, _idx=0):
      self._disp_loading = True
      self.opacity_slider.blockSignals(True)
      self.levels_spin.blockSignals(True)
      self.cs_combo.blockSignals(True)
      ...
      self.opacity_slider.setValue(...)
      self.levels_spin.setValue(...)
      self.cs_combo.setCurrentText(...)
      self.opacity_slider.blockSignals(False)
      self.levels_spin.blockSignals(False)
      self.cs_combo.blockSignals(False)
      self._disp_loading = False

FIX #4: Layer panel should re-read state after Global Settings close
Location: bedrock_window.py (main window)
In the slot that closes Global Settings dialog or after _on_ok:
  self.layer_panel._load_disp_layer()

This ensures the panel UI reflects current state after dialog closes.
